## 1️⃣ Problem — 비즈니스 문제 정의

서울시는 2023년 '펀 라이트 시티 서울' 선언 이후 야간 경제를 핵심 도시경쟁력으로 추진 중이지만, 야간 시간대를 정면으로 다룬 정량·통합 진단은 미흡.

서울시는 야간경제 활성화 정책(청계천 야간명소화, DDP 라이트 페스티벌, 빛초롱축제)과 야간 안전 정책(안심헬프미, 여성안심귀가 스카우트, CPTED, 추가 CCTV, 서울 안전지수)을 양 축으로 확장해 왔다. 그러나 이 정책 도구들은 (1) 시 단위 거점 사업, (2) 자치구 단위 협약·시설 사업, (3) 시 전역 보편 서비스로만 운영되며, 정책이 실제 작동하는 단위인 행정동 수준에서 활성화-안전을 결합한 우선순위 배분 기준이 없다. 동일 자치구 내 행정동 간 야간경제와 안전 격차가 그대로 방치된다.

### 기존 도구의 3가지 한계

- 시간대 해상도: 우리마을가게, 서울 경제관 등은 야간 특화 시간대 분리 불가, 연도별 집계에 그침
- 두 차원의 분리: 경제 잠재력과 안전 인프라가 별도로 평가됨. 통합 진단으로 정책 우선순위를 도출한 사례 부재
- 활용 층위: 현황 공표, 정보 제공 도구만 존재. 행정동 단위 정책 투자 우선순위를 직접 산출하는 의사결정 도구 부재

### 핵심 연구 질문

> "야간경제 정책을 우선 고려해야 할 행정동은 어디인가?"

이 질문을 4단계 하위 질문으로 분해:

- Q1 대상 선별: 야간 상권 정책 적용이 유의미한 행정동은 어디인가?
- Q2 저녁 트랙(17~21시): 잠재력 대비 저녁 매출이 부진한 동은? — 시장 인지 부족 해소 대상
- Q3 심야 트랙(21~06시): 잠재력 + 인프라 공백이 동시에 높은 동은? — 공공재 부족 해소 대상
- Q4 정합성 검정: 정량 신호가 실제 공간(상권축·거리·역세권)에서도 야간경제 거점으로 작동 가능한가?

## 2️⃣ Action — 의사결정 단위별 핵심 처리

### A. 시간대 정의 — 저녁(17~21시) + 심야(21~06시) 분리

- 야간경제의 이론적 기준(18~06시)을 부정하지 않되, 카드매출 데이터의 시간 블록이 17~21시로 제공되는 점을 반영해 분석 범위를 17~06시로 설정

분리의 정량 근거 (EDA → 통계 검정):

- 저녁 카드매출: 2024.06~2025.06 약 3조원대로 급락 후 반등
- 심야 카드매출: 2025.02부터 약 1.9조원대로 점진적 하락
- 2025년 반등은 저녁에만 국한 → 통합 분석이 부적절

윌콕슨 부호순위 검정 (이상치 정제 후 N=387):

- 저녁 성장률 > 심야 성장률: 263/387 (68.0%)
- p < 0.05 (귀무가설 기각)
- 효과 크기 Rank-Biserial R = 0.42 (중간)

> 저녁·심야 분리 접근의 통계적 정당성 확보

### B. 마스터 테이블 설계 — 426 행정동 단일 PK

- 12종 원천 데이터를 행정동 코드 단위로 결합
- 식별/분류/잠재력/인프라/보조 5개 그룹, 트랙별 변수 명시
- 매트릭스 시각 차원과 매핑: X축=잠재력, Y축=인프라 공백(심야)/매출 절댓값(저녁), 점 크기=매출 미실현도, 점 색상=범죄 분위수

### C. 두 가지 지수 설계 — 학술 + 정책 기반

> 가중치 도출 방법론(BAL)과 민감도 분석의 상세는 [docs/methodology.md](docs/methodology.md) 참조.

야간경제 잠재력 지수:

```
0.5 × 매출 성장(YoY) + 0.3 × 점포 개업률 + 0.2 × 교통 유입(YoY)
```

- 50/30/20 강건 (Equal·Entropy 수렴, Spearman ρ ≥ 0.92, Top30 80%↑)

심야 안전 공백 지수:

```
0.45 × CCTV 공백도 + 0.3 × 파출소 접근 취약도 + 0.25 × 보안등 밀도 역수
```

- 45/30/25 강건 (PCA·Equal 수렴, Spearman ρ ≥ 0.97, Top30 95%↑)

심야 트랙 보조 지표:

- 매출 미실현도 (1인당 야간 매출 PercentileRank) → 매트릭스 점 크기(HOW MUCH)
- 범죄 분위수 (자치구 강력+폭력 범죄 합계) → 점 색상(WHEN/WHAT)

### D. 행정동 필터링 파이프라인 — 426 → 108 → 80동

1차 거주지 필터 (426 → 108동):

- 거주 비율 = (주거지역 ∩ 행정동) / 도시 활용 면적
- 도시 활용 면적 = 행정동 면적 – 자연 영역(보전산지 ∪ 개발제한구역 ∪ 도시자연공원)
- 컷오프: 거주 비율 ≥ 67% → 거주지로 정의 (Anderson 1976, USGS Professional Paper 964 토지이용 분류 표준 2/3)
- 수동 보정: 공항동 1건(김포공항 78.5% 점유)만 예외 처리
- 결과: 74.6% 제외 → 108동 통과

2차 상권성 필터 (108 → 80동):

- 데이터: 소상공인진흥공단 상가(상권) 정보 API
- Score = 상권 밀도 × 0.7 + 야간 업종 비중 × 0.3 (음식·주점·카페·오락·편의점·숙박)
- 컷오프: Score 하위 28% 제거 (q28 기준)
- 규모 예외: 상점 개수 ≥ 400인 동은 점수 미달이어도 유지 (양평2동, 발산1동 등 2동)
- 컷오프 강건성: q25/q28/q30 변화 시 후보군 95%+ 반복 유지

### E. 2×2 매트릭스 분석 — 정책 우선 개입형 도출

저녁 트랙 (X: 저녁 잠재력 백분위수 / Y: 2025 저녁 매출 백분위수):

- 우하 분면(잠재력이 높고, 매출이 낮음) = 정책 우선 개입형 16동
- 좌상(이미 활성화) 16동 / 우상(자연 성장 흐름) 25동 / 좌하(기초 여건 개선) 23동

심야 트랙 (X: 심야 잠재력 백분위수 / Y: 인프라 공백도 백분위수):

- 우상 분면(잠재력이 높고, 인프라 공백이 높음) = 정책 우선 개입형 21동
- 점 크기 = 매출 미실현도, 점 색상 = 범죄 분위수로 4차원 시각화

### F. 후보지 정합성 점검 — 매트릭스는 발굴, 정책은 적용

> "매트릭스는 후보를 발굴하지만, 정책은 실제 공간에서 적용된다."

4개 평가 축 (0~2점):

- 야간 허브성: 실제 야간 활동 거점(상권·시장·역세권) 보유 여부
- 적용 범위 명확성: 행정동 전체 vs 특정 거리·시장·역세권 축
- 정책도구 적합성: 기존 서울시 상권·안전 정책과의 결합 가능성
- 지역 특수성: 업무 중심지·백화점·신축 등 일반 골목상권 정책과 구분 필요 여부

평가 기반: 로드뷰, 야간 현장답사 + 평가자 정성 판단

결과: 31동 분석 → 적용 후보 20동 / 보류 5동(도봉2동·자양2동·창4동·잠실6동·고덕2동) / 제외 6동(수서동·여의동·가락1동·염창동·소공동 — 대형시설 의존도 높아 일반 골목상권 정책과 구분 필요)

### G. 기존 정책 도구 연결 — 새 정책이 아닌 우선순위 조정

서울시가 보유한 7개 정책 도구(로컬 브랜드 상권 육성, 골목형 상점가 육성, 전통시장 야간·음식문화 행사, 자치구 단위 골목상권 활성화, 소상공인 종합지원, 우리마을가게 상권분석, 생활폐기물·소음 관리)를 6동 대표 매핑.

| 대표 동 | 트랙 | 유형 | 적용 정책 |
|---|---|---|---|
| 숭인2동 | 저녁 | 권역 연계형 | 자치구 골목상권 활성화 + 전통시장 야간·음식문화 행사 |
| 창1동 | 저녁 | 상권축 중심형 | 골목형 상점가 육성 + 상권축 중심 골목상권 활성화 |
| 문정2동 | 심야 | 업무지구 배후형 | 우리마을가게 + 소상공인 종합지원 + 교통·민원관리 |
| 가양1동 | 심야 | 신흥 업무지구 배후형 | 동일 정책 메뉴, 신규 유입 검증 필요 |
| 용산2가동 | 공통(저녁·심야 양 트랙 1순위) | 골목상권 육성형 | 로컬 브랜드 + 골목형 상점가 + 보행안내·안심귀가 |
| 망원1동 | 공통(양 트랙 2~3순위) | 문화·시장형 안전 관리 | 전통시장 야간 행사 + 생활폐기물·소음 관리 + 보행안내 |

## 3️⃣ Result — 산출물과 인사이트

### 핵심 산출물

- 시간대 분리 진단 매트릭스 (저녁/심야 2트랙)
- 정책 적용 후보 행정동 20동 + 대표 6동 정책 적용표 + 추가 14동 정책 방향표
- 기대효과 KPI 체계 (경제성과 / 공간 정합성 / 안전 민원 / 운영 지속성 / 수용성 가드레일 5층위)
- 비교보정 효과 = 파일럿 권역 변화율 − 비교권역 변화율 (단순 전후 비교가 아닌 인과 추정 프레임)

### 핵심 인사이트

- 저녁: 잠재력은 있으나 매출로 실현되지 않은 공간 → 상권 발견, 홍보, 방문 전환 정책
- 심야: 잠재력은 있으나 안전 인프라 공백이 큰 공간 → 보행 안전, 귀가 동선, CPTED 정책
- 2025년 매출 반등은 저녁에 국한 (윌콕슨 p<0.05, R=0.42 중간 효과) → 시간대 통합 분석은 패턴 왜곡

### 차별점

> "새 정책 개발이 아닌, 서울시가 보유한 정책 도구의 우선순위와 적용 범위 조정"
>
> 비즈니스 발굴이 아닌 공공 정책 Where에 답하는 분석

| 본 연구의 한계 | 후속 연구 방향 |
|---|---|
| 행정동 단위 분석이라 실제 상권축·거리·역세권과 불일치 가능 | 상권축 또는 도로 링크·50m 격자 단위 분석 |
| 정합성 점검이 평가자 정성 판단에 의존 → 주관성 개입 여지 | 복수 평가자 독립 평가 후 Cohen's Kappa로 일치도 검증 |
| 일부 안전·범죄 데이터가 자치구 단위로만 공개되어 행정동 차이 직접 반영 어려움 | 향후 112·120 민원, 동 단위 신고자료, 추가 현장답사로 행정동 내부 검증 |
| 안전 인프라 지수는 공급 측 지표(수량)로 체감 안전도 미반영 | 주민·상인 설문, 체감안전도, CCTV 작동 여부 반영 |
| 정책 개입 시 야간 수요 증가가 신규 창출인지, 주간→야간 이전인지 혹은 지역간 단순 수요 이동인지 구분 불가 | 미시행 인접 동을 통제군으로 신규 창출 vs 공간 대체율 정량화, 실험설계를 통해 야간 수요를 극대화하는 요인 탐색 |
| 신축·개발지는 현재 데이터 왜곡 가능 | 1~2년 단위 모니터링 후보군으로 재측정 |
| 가중치는 학술 + 정책 중요도 기반이나 최종 수치엔 연구자 정책 판단 포함 | 실제 정책성과 데이터 축적 후 가중치 임곗값 재조정 |

> 본 연구의 20동은 최종 사업 확정지가 아닌, 서울시·자치구가 기존 정책 도구를 우선 검토할 수 있는 사전 진단 후보지로서 활용.

## 재현 가능성

**기본 원칙**: raw 데이터 + 분석 코드만 있으면 처음부터 최종 산출물까지 완전 재현 가능하도록 설계.

**예외 — 캠퍼스 반출 제약**:
- B013·B040·B079는 서울 빅데이터 캠퍼스 내부 데이터로, 본 리포지토리에 raw가 포함되어 있지 않다.
- 0_preprocess 단계 중 b013·b040·b079는 캠퍼스 환경에서만 실행 가능.
- 외부 환경에서는 `data/processed/` 의 가공 산출물부터 재현 가능.
- B079 원본은 외부 반출 불가이므로, 외부 검증 시 카드매출 변수는 OA-22175(공개 추정매출-행정동)로 대체된다.

**모든 모듈 입력 default 경로** = raw 위치 또는 선행 모듈의 정확한 산출 경로. 27개 모듈 출력→입력 매칭 검증 0건 실패.

**검증된 raw 시작점** (사용자가 raw만 있으면 실행 가능한 모듈):
- 캠퍼스 내부: b013.py, b040.py, b079.py (E:/ 캠퍼스 raw 경로)
- 캠퍼스 외부: opening_rate.py·crime_context.py·cctv_void.py·police_void.py·light_void.py·light_geocode.py·residential.py·commerce.py·evening_sales.py
- 모든 코드 파일에는 캠퍼스 반출 정책 준수를 위해 `print()`, `display()`, `.head()` 등 데이터 확인 출력이 제거되어 있다.
